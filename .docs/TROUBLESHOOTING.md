# 트러블슈팅 이력 (Troubleshooting)

프로젝트 진행 중 발생하는 주요 버그, 아키텍처 결함, 및 Docker(M1/M2) 관련 환경 오류에 대한 원인 분석 및 해결 과정을 기록함.

## 작성 규칙

1. **문제 현상 (Symptom)**: 구체적인 에러 메시지 포함.
2. **원인 분석 (Root Cause)**: 예: `Node.js` 네이티브 모듈 컴파일 실패, M1 아키텍처 호환성 등.
3. **해결 과정 (Solution)**: 구체적인 Docker 명령어 및 코드 변경 내역 서술(개조식).
4. **결과 (Result)**: 검증된 최종 정상 작동 상태 기술.
